## 3. Three High-Impact Improvements

### 3.1 Speak fluently in tool features

**Asana sample answer:**
> Project as a board with columns: *Backlog → Scoped → In Progress → Blocked → Review → Done*. Custom fields: Owner, Priority (P0–P3), Due Date, Status (R/A/G), Effort (S/M/L), Dependency. Use Timeline view for critical path, Workload view for over-allocation. Rules automate Slack pings on Blocked. Project templates standardize new client engagements.

Build equivalent answers for:
- **Monday.com:** Items, sub-items, Status columns, Mirror columns, Automations, Workload, Dashboards
- **Basecamp:** Message Board, To-dos with assignee/due date, Schedule, Automatic Check-ins, Hill Charts, Pings

**Action:** Free trial each tool. Build the same sample project in all three. 4–6 hrs.

### 3.2 Quantify risk management

> Weekly RAG cadence with explicit thresholds:
> - 🟢 **Green:** on/ahead of schedule, <5% scope variance, no open P0/P1
> - 🟡 **Amber:** 1–10% schedule slip OR 1 open P1 OR one dep at risk → PM owns recovery plan within 48 hrs
> - 🔴 **Red:** >10% slip OR P0 open OR budget variance >15% → escalate to sponsor in 24 hrs, daily standups until Amber
>
> RACI: PM is Accountable for status, Responsible for recovery; sponsor Consulted on Red; exec Informed weekly.

### 3.3 Standard documentation template

```
**Project:** <name>     **Week of:** <date>
**Status:** 🟢 / 🟡 / 🔴
**Progress this week:**
  - <Bullet 1>
  - <Bullet 2>
**Blockers / Risks:**
  - <Issue> | Owner | Mitigation | Target resolution
**Decisions made:**
  - <Decision> | Decided by | Date
**Next week's priorities:**
  - <Bullet 1>
  - <Bullet 2>
**Confidence in hitting next milestone (1–5):** <n>
**Last updated:** <timestamp + author>
```

The timestamp + confidence score makes it auditable; over time, plot confidence vs actual delivery to calibrate optimism per owner.
